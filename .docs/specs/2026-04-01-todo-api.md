# Design Doc: Todo API

**Date:** 2026-04-01  
**Status:** Approved  
**Approach:** A — Minimal (2 endpoints)

---

## Problem

Need a simple REST API to manage todos. No authentication, no users. A todo can be created and
marked as done. Nothing else.

---

## Solution

A Rails 7 API-only application with a single `Todo` model and two endpoints:

| Method | Path | Action |
|--------|------|--------|
| `POST` | `/api/todos` | Create a new todo |
| `PATCH` | `/api/todos/:id/complete` | Mark a todo as done |

Controller namespaced under `Api::TodosController`, following project conventions.

### Todo Model

| Field | Type | Notes |
|-------|------|-------|
| `title` | string | Required, the todo text |
| `done` | boolean | Default: false |
| `created_at` | datetime | Auto-managed |
| `updated_at` | datetime | Auto-managed |

### Endpoint Details

**POST /api/todos**
- Body: `{ "todo": { "title": "Buy milk" } }`
- Success: `201 Created` with todo JSON
- Failure: `422 Unprocessable Entity` if title is blank

**PATCH /api/todos/:id/complete**
- No body required
- Success: `200 OK` with updated todo JSON
- Failure: `404 Not Found` if id does not exist

---

## Scope

### In
- `Todo` model with `title` (string) and `done` (boolean)
- `POST /api/todos` — create
- `PATCH /api/todos/:id/complete` — mark done
- RSpec request specs for both endpoints
- FactoryBot factory for `Todo`

### Out
- Listing todos (`GET /api/todos`)
- Showing a single todo (`GET /api/todos/:id`)
- Deleting todos (`DELETE /api/todos/:id`)
- Editing/updating title
- Marking a todo as not-done (undo)
- Authentication / users
- Pagination, filtering, sorting

---

## Key Decisions

1. **`/complete` sub-resource over `PATCH /api/todos/:id`** — A dedicated route makes the
   intent explicit and avoids a general-purpose update endpoint that wasn't requested.

2. **`done` is boolean, not a status enum** — The only state transition requested is
   "mark as done." An enum would be over-engineering.

3. **No `GET` endpoints** — Strictly in scope per Approach A. Manual testing can use the
   `id` returned from `POST`.

4. **Namespace `Api::`** — Matches project conventions in CLAUDE.md.
